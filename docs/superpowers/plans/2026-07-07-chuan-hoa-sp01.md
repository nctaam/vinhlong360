# SP0+SP1 Chuẩn-hoá Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: superpowers:executing-plans (inline). Steps dùng checkbox. 1 việc/commit, nhánh `feat/chuan-hoa-sp01`.
> **STATUS (2026-07-07): active — đang thực thi.** Spec nguồn: `docs/superpowers/specs/2026-07-07-chuan-hoa-program-design.md` (v2 khắt khe, đã duyệt).

**Goal:** Ban hành bộ tiêu chuẩn `docs/standards/` (8 file, mọi rule có cách đo) + cơ chế ép hai tầng có RATCHET (12 check-module, pre-commit hook <5s, scorecard điểm-không-tụt, pre_merge mở rộng).

**Architecture:** `scripts/checks/common.py` là nền declarative — CheckResult chuẩn, helper staged-files, `RegexCheck` factory và baseline IO; 8/12 module chỉ là config khai báo, 4 module (data_schema, api_contract, links, test_pairing) có logic riêng. Ratchet = so `count` với `docs/standards/baseline.json` (committed). Scorecard đọc mọi check → điểm/chiều → append `scorecard-history.jsonl`, chặn tụt điểm.

**Tech Stack:** Python 3.14 stdlib (re, json, subprocess, pathlib, ast), pytest; hook sh cho Git-Bash Windows. KHÔNG dependency mới (B8).

## Global Constraints (từ spec v2)

- RATCHET: mọi counter tăng so baseline = chặn (kể cả tầng soft-ratchet); baseline chỉ được cập nhật trong cùng commit với thay đổi diện-rộng có giải trình.
- KHÔNG skip lớp hard; `SKIP_CHECKS` chỉ soft + bắt buộc `SKIP_REASON` + append `docs/standards/90-exceptions-log.md`.
- Hook <5 giây (test đo); chỉ chạy trên staged files.
- 100% check-module có test; TDD từng module.
- Rule không máy-đo-được không được ban hành (trừ ngoại lệ ký từng-rule).
- Không phá bất biến CLAUDE.md §2; không dependency/dịch vụ mới.
- Scorecard: 0-100/chiều, đích ≥95, sàn 90; điểm tụt = pre_merge chặn.
